# Documentation Map

This folder is for quick onboarding and practical reference while you experiment.

- `quickstart.md`: install, prepare data, train, and inspect outputs.
- `architecture.md`: how code is split between data, model, and training.
- `ml-fundamentals.md`: key concepts used in this project.
- `data-pipeline.md`: supported dataset types and token-file workflow.
- `training.md`: training loop, checkpoints, and resume behavior.
- `testing.md`: how to run tests and what each test scope validates.

Recommended reading order for beginners:

1. `quickstart.md`
2. `architecture.md`
3. `ml-fundamentals.md`
4. `data-pipeline.md`
5. `training.md`
6. `testing.md`
